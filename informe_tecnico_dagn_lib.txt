================================================================================
2. CONTEXTO Y MOTIVACIÓN
================================================================================

DAGN (Doctoral Affective Generalization Network) investiga el reconocimiento
automático de emociones en el espacio de Valencia/Arousal (VA) de Russell (1980).

Problema central: fusionar señales heterogéneas (vídeo facial, fisiología, EEG)
de múltiples datasets con convenciones distintas de anotación.

Elección arquitectónica: frente a encoders profundos (dagn_simple), dagn_lib
apuesta por features explícitas extraídas con librerías validadas en la literatura.
Esta elección es el núcleo del argumento doctoral:

  "Features explícitas extraídas con herramientas establecidas + modelo minimalista
   + prior diferenciable desde literatura canónica = máxima justificabilidad
   con mínima complejidad paramétrica"

Datasets involucrados (5 total):
  - DEAP:    EEG 32ch + BVP/GSR/TEMP (laboratorio, 32 sujetos, 40 videos)
  - WESAD:   BVP/EDA/TEMP de muñeca (Empatica E4, estrés/no estrés)
  - DREAMER: EEG 14ch + ECG (Emotiv EPOC, 23 sujetos, 18 películas)
  - AFEW-VA: Vídeo de expresión facial en-the-wild (clips AFEW, VA 0-10)
  - AFFEC:   EEG 63ch + GSR + OpenFace2 AUs (72 sujetos, 4 emociones)

4. ARQUITECTURA DEL MODELO
================================================================================

Archivo: production/fusion_model.py
Clase:   FusionLSTM (nn.Module)

── 4.1 Diagrama de arquitectura ────────────────────────────────────────────────

  Entrada:
    face   (B, T=30, 17)  — Action Units MediaPipe FACS [0,1]
    physio (B, T=30,  6)  — HRV/EDA/TEMP NeuroKit2 (normalizado)
    eeg    (B, T=30,  5)  — TGAM2-compatible log1p [0, 0.405]

  ┌─────────────────────────────────────────────────────────────┐
  │  face[B,T,17] ──┐                                           │
  │  physio[B,T,6] ─┼─ cat → [B,T,28]                          │
  │  eeg[B,T,5]  ───┘                                           │
  │                                                             │
  │  [B,T,28] → LayerNorm(28) → [B,T,28]                        │
  │                                                             │
  │  [B,T,28] → LSTM(28→256, L=2, inter_drop=0.2) → [B,T,256]  │
  │                                                             │
  │  [B,T,256] → Dropout(0.45) → [B,T,256]                     │
  │                                                             │
  │  [B,T,256] → Linear(256→2) → tanh → va [B,T,2]             │
  │                                                             │
  │  grad[:,0,:] = 0                                            │
  │  grad[:,t,:] = va[:,t,:] - va[:,t-1,:]  (t>0)              │
  └─────────────────────────────────────────────────────────────┘

  Salida:
    va   (B, T, 2)  — valence/arousal ∈ [-1, 1] por timestep
    grad (B, T, 2)  — gradiente temporal de VA (trayectoria emocional)

── 4.2 Recuento de parámetros (verificado) ─────────────────────────────────────

  Capa              Parámetros    Cálculo
  ─────────────────────────────────────────────────────
  LayerNorm(28)          56       2 × 28 (weight + bias)
  LSTM L1 (28→256)  292,864       4 × (256×(28+256) + 2×256) = 4 × 73,216
  LSTM L2 (256→256) 526,336       4 × (256×(256+256) + 2×256) = 4 × 131,584
  Linear(256→2)         514       256×2 + 2
  ─────────────────────────────────────────────────────
  TOTAL             819,770       ≪ 8,800,000 dagn_simple  (≈10× reducción)

── 4.3 Decisiones de diseño ────────────────────────────────────────────────────

  1. LayerNorm antes del LSTM:
     Normaliza las escalas heterogéneas entre modalidades.
     Face AU está en [0,1], EDA está z-scored (puede ser ±3), EEG en [0,0.405].
     LayerNorm equipara la magnitud antes de entrar al LSTM.

  2. LSTM en lugar de Transformer:
     El LSTM es más eficiente para secuencias cortas (T=30).
     Captura la dinámica temporal de las emociones sin la sobrecarga
     de mecanismos de atención. Interpretable a nivel de "estado oculto".

  3. Forget gate bias = 1:
     _init_weights() inicializa el bias de forget gate a 1.0 para
     mejorar el flujo de gradientes y evitar el vanishing gradient
     en las primeras épocas (Gers et al. 2000).

  4. inter_dropout = 0.2 entre capas LSTM:
     Activado solo cuando num_layers > 1 (caso actual: 2 capas).

  5. tanh en salida:
     Acota VA a [-1, 1], consistente con la normalización z-score
     de los labels (clipeados a ±3 std, la mayoría en [-1,1]).

  6. Gradiente temporal:
     La salida grad se usa en producción para detectar cambios bruscos
     de estado emocional. No contribuye al loss durante entrenamiento.

── 4.4 Hiperparámetros finales ─────────────────────────────────────────────────

  hidden_dim  = 256
  num_layers  = 2
  dropout     = 0.45     (post-LSTM output)
  inter_drop  = 0.2      (between LSTM layers)
  T           = 30       (timesteps por sample)
  INPUT_DIM   = 28       (17 + 6 + 5)

── 5.1 Features de cara — face (T, 17) ─────────────────────────────────────────

Archivo:   training/feature_extractor_face.py
Biblioteca: MediaPipe FaceMesh (468 landmarks, static_image_mode=True)
Referencia: Ekman & Friesen (1978) Facial Action Coding System (FACS)

Método: geometric approximation (no regresión de AUs).
Cada AU se computa como una relación geométrica entre landmarks,
normalizada por distancia inter-ocular y altura facial.

Tabla de AUs extraídas:
  Idx │ AU   │ Descripción                    │ Cálculo (resumen)
  ────┼──────┼────────────────────────────────┼──────────────────────────────
    0 │ AU01 │ Inner Brow Raise               │ inner_brow_y relativo a eye_center
    1 │ AU02 │ Outer Brow Raise               │ outer_brow_y relativo a eye_center
    2 │ AU04 │ Brow Lowerer                   │ inner_brow descent hacia nasal bridge
    3 │ AU06 │ Cheek Raiser                   │ cheek_y relativo a nose_y (smile cheek)
    4 │ AU07 │ Lid Tightener                  │ max(0, 0.30 - EAR_avg) × 5
    5 │ AU10 │ Upper Lip Raiser               │ nose_y − mouth_top_y normalizado
    6 │ AU12 │ Lip Corner Puller              │ corner raise sobre mouth_mid_y
    7 │ AU14 │ Dimpler                        │ mouth_width / face_width reducido
    8 │ AU15 │ Lip Corner Depressor           │ corner drop bajo mouth_mid_y
    9 │ AU17 │ Chin Raiser                    │ chin_y hacia lower_lip
   10 │ AU20 │ Lip Stretcher                  │ mouth_width / face_width
   11 │ AU23 │ Lip Tightener                  │ inter-lip gap reducido (<0.06)
   12 │ AU24 │ Lip Pressor                    │ near-contact labial (<0.03)
   13 │ AU25 │ Lips Part                      │ inter-lip gap vertical × 8
   14 │ AU26 │ Jaw Drop                       │ |mouth_bot − mouth_top| / face_height × 5
   15 │ AU28 │ Lip Suck                       │ lip_gap muy reducido (<0.04)
   16 │ AU43 │ Eyes Closed (EAR)              │ max(0, 0.25 − EAR_avg) × 5

Rango de salida: [0.0, 1.0] por AU.
Referencia de normalización: IOD (distancia inter-canthi, lm33↔lm263).
Face detection failure → zeros(17) para ese frame.

Para AFFEC: se usan los AU_r reales de OpenFace2 en [0,5] almacenados en .npz.
            El prior ajusta la escala AFFEC /5 para que las reglas sean comparables.
Para DEAP/WESAD/DREAMER: zeros(T, 17) (no tienen vídeo).

── 5.2 Features fisiológicas — physio (T, 6) ───────────────────────────────────

Archivo:    training/feature_extractor_physio.py
Biblioteca: NeuroKit2 (Makowski et al. 2021, Behavior Research Methods)
Referencias: Task Force ESC/NASPE (1996), Boucsein (2012), Gross & Levenson (1993)

  Col │ Feature       │ Cálculo / normalización
  ────┼───────────────┼────────────────────────────────────────────────────────
    0 │ RMSSD_norm    │ RMSSD(ms) / 100  (raíz de diferencias RR al cuadrado)
    1 │ SDNN_norm     │ SDNN(ms) / 100   (desviación estándar intervalos NN)
    2 │ mean_HR_norm  │ HR(BPM) / 100    (frecuencia cardíaca media)
    3 │ EDA_tonic     │ SCL decomp NeuroKit2, z-score dentro del trial
    4 │ EDA_phasic    │ SCR decomp NeuroKit2, z-score dentro del trial
    5 │ TEMP_norm     │ (°C − 34) / 4    (temperatura periférica centrada en 34°C)

HRV (cols 0-2): estadísticos de ventana completa, replicados en todos los T timesteps.
  Proceso: BVP/ECG → NeuroKit2 ppg_process/ecg_process → picos → RR intervals
           Filtro fisiológico: solo RR ∈ [300, 2000] ms (30-200 BPM) son válidos
EDA (cols 3-4): descompuesta por NeuroKit2, mean-pooling a T timesteps, z-scored.
TEMP (col 5): mean-pooling a T timesteps, normalización lineal.

Rango típico: RMSSD 0.15-0.80, HR 0.60-0.95, EDA ±2 (z-scored), TEMP ±0.5.

── 5.3 Features EEG — eeg (T, 5) ───────────────────────────────────────────────

Archivo:    training/feature_extractor_eeg_tgam2.py
Biblioteca: MNE / scipy.signal (Welch PSD dentro de feature_extractor_eeg.py)
Referencias: Klimesch (1999), Crowley (2010), Davidson (1988)

Pipeline EEG (entrenamiento):
  EEG multichannel → canales frontales (F3/F4) → bandpower Welch (1s ventana)
    → theta_raw, alpha_raw, beta_raw
    → att = clip(beta/(alpha+theta+ε)×50, 0, 100)
    → med = clip(alpha/(theta+beta+ε)×50, 0, 100)
    → _eeg_approx(att, med) → 5 features

Pipeline EEG (producción):
  NeuroSky TGAM2: att, med (0-100) directamente
    → _eeg_approx(att, med) → 5 features (idéntico)

Formula _eeg_approx(att, med):
  theta = log1p((1 − att/100) × 0.5)   — relaxación/inatención
  alpha = log1p((med/100) × 0.5)        — meditación/relajación alertada
  beta  = log1p((att/100) × 0.5)        — atención/carga cognitiva
  asym  = 0.0                            — TGAM2 mono-frontal (no asimetría)
  ratio = theta − alpha                  — índice theta/alpha Klimesch

Rango: [0, log1p(0.5)] ≈ [0, 0.405] — acotado, no z-scored, igual en train/infer.

Canales frontales por dataset:
  DEAP (32ch, 128Hz):   F3=ch2, F4=ch19
  DREAMER (14ch, 128Hz): F3=ch2, F4=ch11  (Emotiv EPOC)
  AFFEC (63ch, 256Hz):   zeroed (EEG excluido — canal incertidumbre)
  AFEW-VA, WESAD:        zeros (no EEG)

Bandas de frecuencia:
  Theta: 4-8 Hz, Alpha: 8-13 Hz, Beta: 13-30 Hz
  Calculadas con Welch PSD (from feature_extractor_eeg.py)

7. PIPELINE DE DATOS Y NORMALIZACIÓN
================================================================================

Archivo: training/global_dataset.py

── 7.1 Construcción del dataset global ─────────────────────────────────────────

  1. Carga de subdatasets: DEAPDataset, WESADDataset, DREAMERDataset(None),
     AFEWVADataset, AFFECDataset → lista de (face, physio, eeg, va, ds_id)
  2. Split estratificado: 80% train / 20% val dentro de cada dataset (seed=42)
  3. VA z-score: media y desviación estándar calculadas SOLO sobre split train
     → aplicadas a val usando las mismas estadísticas (sin data leakage)
     → labels clipeadas a ±3 std para eliminar outliers
  4. Split "test" (no usado en producción): primeros 50% de índices val de
     cada dataset — selección indirecta vía early stopping (bias leve)

── 7.2 Sampler ponderado ────────────────────────────────────────────────────────

  WeightedRandomSampler:
    weight[i] = quality_weight[ds_id] / sqrt(n_ds)

  quality_weights = {DEAP: 1.0, WESAD: 1.0, DREAMER: 0.0, AFEW-VA: 1.0, AFFEC: 1.0}
    → Todos los datasets activos tienen igual importancia (equal quality)
    → 1/sqrt(n) equilibra tamaños dispares (WESAD≈1498 vs DREAMER=0)

── 7.3 VA z-score por dataset ───────────────────────────────────────────────────

  El z-score es la solución al "dataset bias problem":
  - DEAP usa escala 1-9, WESAD usa arousal discreto, AFEW-VA usa 0-10.
  - Sin normalización, el modelo aprende los offsets inter-dataset
    en lugar de la dinámica emocional intra-dataset.
  - Con z-score por dataset, cada uno contribuye con mean=0, std=1,
    y el modelo aprende el ranking relativo dentro de cada dataset.

  Implicación para el prior: las reglas absolutas (AU06>0.3 → V=+0.65)
  se aplican a un espacio z-scored donde +0.65 puede corresponder a
  distintos valores absolutos según el dataset. Por eso el prior usa
  "directional hinge" (ver sección 9) en lugar de targets absolutos.

── 7.4 Modal dropout augmentation ──────────────────────────────────────────────

  Durante entrenamiento, cada muestra tiene probabilidad de que
  una modalidad completa sea zeroed out:
    P_FACE_DROP   = 0.2  (20% de muestras pierden toda la cara)
    P_PHYSIO_DROP = 0.2  (20% pierden toda la fisiología)
    P_EEG_DROP    = 0.3  (30% pierden todo el EEG)

  Objetivo: el modelo aprende a predecir incluso con modalidades ausentes,
  lo que es necesario porque en producción real solo hay physio+EEG (no vídeo).

8. PROCESO DE ENTRENAMIENTO
================================================================================

Archivo: training/train_fusion.py
Comando: python -u train_fusion.py --use_prior

── 8.1 Hiperparámetros ──────────────────────────────────────────────────────────

  EPOCHS      = 200
  BATCH_SIZE  = 32
  LR          = 1e-3      (AdamW)
  WEIGHT_DECAY= 3e-4      (regularización L2)
  PATIENCE    = 60        (early stopping — *nota: docstring dice 40, es erróneo*)
  T           = 30        (timesteps por sample)
  SWA_START   = 80        (inicio de Stochastic Weight Averaging)
  P_FACE_DROP = 0.2       (modal dropout cara)
  P_PHYSIO_DROP= 0.2      (modal dropout fisiología)
  P_EEG_DROP  = 0.3       (modal dropout EEG)

── 8.2 Función de pérdida total ────────────────────────────────────────────────

  L = MSE(pred_seq, target_rep) + (1 - CCC_V) + (1 - CCC_A) + variance_penalty + λ·L_prior

  MSE: error cuadrático medio entre predicción temporal y target replicado.
       pred (B,T,2) vs target_rep (B,T,2) donde target_rep[:, t, :] = target

  CCC loss: 1 - CCC sobre todas las muestras del batch.
       Mide acuerdo de concordancia (Lin 1989), robusto al offset de media.
       Calculado por separado para valence y arousal, promediado.

  Variance penalty: penaliza predicciones planas (std_pred < std_target).
       penalty = α × relu(std_target − std_pred), por dataset.
       α = {DEAP: 0.5, WESAD: 0.0, DREAMER: 0.0, AFEW-VA: 0.3, AFFEC: 0.3}
       WESAD=0.0: la fisiología de muñeca ya induce varianza suficiente.
       DEAP=0.5: señal EEG + physio tiende a predicciones planas.

  λ·L_prior: prior fisiológico diferenciable (sección 9). λ=0.05.

── 8.3 Scheduler ────────────────────────────────────────────────────────────────

  CosineAnnealingLR(T_max=200, eta_min=LR×0.01=1e-5)
  → LR decae de 1e-3 a 1e-5 siguiendo coseno a lo largo de EPOCHS.

── 8.4 Gradient clipping ────────────────────────────────────────────────────────

  nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
  → Previene exploding gradients con LSTMs profundos.

── 8.5 Stochastic Weight Averaging (SWA) ───────────────────────────────────────

  Algoritmo: Izmailov et al. (2018) "Averaging Weights Leads to Wider Optima
  and Better Generalization".

  Implementación:
    swa_model = AveragedModel(model)
    for epoch >= SWA_START: swa_model.update_parameters(model)
    Al final: si swa_model mejora best_ccc → guardar swa checkpoint

  Observación en ablation: SWA ayuda cuando el best epoch cae en medio del
  rango de averaging. Con best_ep=115 y SWA_START=80, el escenario es ideal.
  En la configuración final, el SWA no superó el checkpoint regular.

── 8.6 Early stopping ───────────────────────────────────────────────────────────

  Métrica: mean_ccc = (CCC_V_global + CCC_A_global) / 2  sobre val set.
  Checkpoint guardado en production/fusion_best.pth cuando mejora.
  Epoch final del mejor resultado: epoch 115, early stopping en epoch 175.

── 8.7 Evaluación durante entrenamiento ────────────────────────────────────────

  eval_ccc_by_dataset(): eval en modo no-grad, usa último timestep (t=-1).
  Imprime por consola: epoch, train_loss, CCC-V, CCC-A, mean, per-dataset.
  Registra en results_log.txt al finalizar.

── 8.8 Diagnóstico go/no-go (epoch 30) ─────────────────────────────────────────

  Si en el epoch 30 AFFEC arousal < -0.10:
    → El run cayó en un mínimo local malo (AFFEC stuck negativo).
    → Matar el proceso y reiniciar (sin seed=42 para diferente init).
  Causa: el prior con reglas EEG/physio en ciertos basins de inicialización
  induce un sesgo sistemático en AFFEC arousal.

9. PRIOR FISIOLÓGICO DIFERENCIABLE
================================================================================

Archivo: training/physiological_prior.py
Clase:   PhysiologicalPrior(nn.Module)

── 9.1 Motivación ───────────────────────────────────────────────────────────────

El prior encoda conocimiento experto de la literatura como restricciones
diferenciables durante el entrenamiento. No es un módulo de inferencia;
es una pérdida auxiliar que guía el modelo hacia predicciones que respetan
las relaciones señal→emoción documentadas en la literatura.

Dirección de las reglas: observed_signal → expected_VA_range
  (NO emotion → signal; la dirección inversa es la que el modelo necesita)

── 9.2 Arquitectura del prior ───────────────────────────────────────────────────

  compute_prior_va(face, physio, eeg, ds_ids):
    → Por cada muestra, aplica N reglas condicionadas a la modalidad presente
    → Cada regla devuelve (v_vote, a_vote, weight)
    → Weighted voting: v_prior = Σ(v_i × w_i) / Σ(w_i)
    → Sigma = sigma_global / (1 + Σw_i)  → decrece con más evidencia

  auxiliary_loss(pred_va, face, physio, eeg, ds_ids):
    → Hinge direccional: relu(-pred_v × sign(v_mu)) × v_active
    → Fires solo cuando: n_rules > 0.1 AND |mu| > 0.15
    → L = λ × mean(loss_v + loss_a) sobre el batch

  Zero-masking: si physio está completamente en cero (modal dropout o
  dataset sin fisiología), las reglas de physio NO se aplican.
  Igual para face y eeg.

── 9.6 Pérdida auxiliar: directional hinge ─────────────────────────────────────

  Diseño: Gaussian NLL fue probado inicialmente y falló (CCC cayó a 0.138).
  El problema: targets absolutos (V=+0.65 para Duchenne) son incompatibles
  con labels z-scored donde el mismo V puede ser negativo si el clip está
  bajo la media del dataset.

  Solución: directional hinge — solo pregunta "¿está la predicción en la
  mitad correcta del espacio?"

    L_v = relu(-pred_v × sign(v_mu)) × gate_v
    L_a = relu(-pred_a × sign(a_mu)) × gate_a

  gate = (n_rules > 0.1) × (|mu| > 0.15)
  → Activa solo cuando el prior tiene señal unívoca.

    - Reglas physio y EEG con defaults (DEAP, WESAD, AFFEC para physio;
      DEAP, DREAMER para EEG — aunque DREAMER excluido del training).
    - λ = 0.05 (no penalizar demasiado; el modelo tiene libertad de adaptarse).

── 10.1 Metodología de evaluación ──────────────────────────────────────────────

  Split: val (80/20 estratificado, seed=42)
         Val = muestras que nunca recibieron actualizaciones de gradiente.
         NOTA: Early stopping usó el val set completo → model selection bias.
         Para test sin sesgo requeriría re-entrenar con split 70/15/15.

  Métrica principal: CCC (Concordance Correlation Coefficient, Lin 1989)
    CCC(p,t) = 2·cov(p,t) / (var(p) + var(t) + (mean_p − mean_t)²)
    CCC ∈ [-1, 1]; CCC=1 es acuerdo perfecto.
    Ventaja sobre Pearson r: penaliza además la diferencia de medias.

  Intervalo de confianza 95%: bootstrap percentil (n=1000 resamples, seed=42).

  Inferencia: último timestep (t=-1) → máximo contexto temporal.
  Consistente con producción.

  Métricas adicionales (evaluate_fusion --split val --verbose):
    Pearson r, p-valor, RMSE, MAE, R²

15. ARGUMENTACIÓN DOCTORAL
================================================================================

dagn_lib encarna la hipótesis doctoral central: las features explícitas
extraídas con herramientas bibliográficamente fundamentadas son más
justificables y, en dominios con datos etiquetados limitados, pueden
competir con encoders profundos con una fracción de sus parámetros.

── 15.1 Contribuciones técnicas ────────────────────────────────────────────────

  1. Pipeline de features multimodal explicable:
     - Face: 17 AUs geométricos desde MediaPipe (Ekman 1978 FACS)
     - Physio: 6 features HRV/EDA/TEMP desde NeuroKit2 (Task Force 1996)
     - EEG: 5 features TGAM2-compatible desde MNE (Klimesch 1999, Crowley 2010)
     Cada feature tiene una referencia canónica en la literatura.

  2. Prior fisiológico diferenciable:
     Sistema de reglas señal→VA derivadas de literatura (Ekman 1978, Task Force
     1996, Boucsein 2012, Klimesch 1999) expresadas como pérdida auxiliar.
     Primer prior diferenciable de este tipo en el sistema DAGN.

  3. Diseño TGAM2-compatible:
     El espacio de features EEG es idéntico entre entrenamiento (multichannel)
     y producción (sensor NeuroSky TGAM2 de bajo coste).
     Elimina el distribution shift entrenamiento/inferencia para EEG.

  4. Ablation exhaustivo documentado:
     5 variantes del prior, EEG bilateral, SWA, seed fija.
